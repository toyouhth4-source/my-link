## 1. 프로젝트 개요
*   **프로젝트명:** 마이링크 (MyLink) - 개인 및 지정 그룹용
*   **목적:** '나 자신'을 가장 잘 소개하고 증명할 수 있는 고품질 프라이빗 프로필/포트폴리오 사이트.
*   **디자인 컨셉:** **네오 브루탈리즘 (Neo-Brutalism)** 
*   **열람 정책:** 일반 브라우저 접속은 허용하되, 구글 수집 태그(noindex) 삽입으로 검색 엔진 노출 차단.

---

## 2. 핵심 기능 목록 (개발자 포트폴리오 중심)

### 2.1 필수 기능 (Required)
*   **[P1] '나'를 소개하는 핵심 프로필 뷰**
*   **[P1] 아카이브 보드 및 다중 링크** (프로젝트 갤러리 형태로 구성)
*   **[P1] 테마 및 디자인 전면 적용** (라이트 모드: 크림/원색 기반 네오 브루탈리즘, 다크 모드: 네온/형광 기반 사이버펑크 콘셉트로 스위칭)
*   **[P1] 폐쇄형 뼈대 시스템 통합 (Firebase)** 
    *   **Auth:** 구글 소셜 로그인 연동 (방명록 작성용)
    *   **DB:** Firebase Firestore를 채택하여 방문자 방명록 댓글과 링크 데이터를 고속으로 저장/조회.

### 2.2 심화 및 고급 UX 기능 (Advanced)
*   **[P2] Google 생태계 통합 간소화** (Drive 공유 링크 다이렉트 매핑으로 딜레이 없는 이력서 다운로드, Analytics 방문 통계)
*   **[P3] Github Contribution(잔디) 시각화 위젯** 
*   **[P3] 키보드 단축키(Cmd/Ctrl + K) 기반 커맨드 팔레트** 
*   **[P3] 기술 스택 무빙 텍스트(Marquee) 애니메이션** 
*   **[P4] 프라이빗 컨셉을 살린 위트 있는 404/보안 거부 페이지**
*   **[P4] 소소한 마이크로 물리적 인터랙션** (클릭 시 푹 꺼지는 모션 등)

## 3. 구조적 아키텍처 및 정책
*   **드라이브 고속 연동:** API 통신에 소모되는 시간을 완전히 없애기 위해, 구글 드라이브 리소스의 공유(Public) URL 주소를 직접 바인딩하여 0.1초의 로드 딜레이도 발생하지 않도록 함.
*   **SSG/ISR 캐싱 + Firestore:** 트래픽이 몰려도 Firestore 읽기 비용이 청구되지 않도록 Next.js의 웹 캐시 시스템과 결합.
