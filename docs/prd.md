## 1. 프로젝트 개요

*   **프로젝트명:** 마이링크 (MyLink) - 개인 및 지정 그룹용
*   **프로젝트 목적:** '나 자신(개발자/학생)'을 가장 잘 소개하고 증명할 수 있는 고품질 통합 프로필 사이트. 과제나 프로젝트를 직관적으로 보여줌과 동시에, 내 사이트에 접근이 허락된 지인(교수님, 학우들)과 소통할 수 있는 폐쇄형(지인 기반) 포트폴리오 아지트 역할.
*   **디자인 컨셉:** **네오 브루탈리즘 (Neo-Brutalism)** - 두꺼운 테두리, 원색 대비, 깊은 그림자를 활용하여 개발자스러운 개성과 힙한 에너지를 동시에 발산하는 디자인.
*   **열람 정책:** 
    *   누구나 일반 웹 브라우저를 통해 접속할 수 있습니다 (포트폴리오 열람 목적).
    *   단락별 검색엔진 노출을 막아(noindex) 개인 프라이버시를 유지합니다.

## 2. 핵심 기능 목록 (개발자 포트폴리오 중심)

### 2.1 필수 기능 (Required)
*   **[P1] '나'를 소개하는 핵심 프로필 뷰** (사진, 전공, 학번 등 개인 아이덴티티 부각)
*   **[P1] 아카이브 보드 및 다중 링크** (프로젝트 갤러리 형태의 컴포넌트로 깃허브, 블로그 등 연결)
*   **[P1] 네오 브루탈리즘 테마 전면 적용** (UI/UX 뼈대 전체에 굵은 선형 디자인 반영)
*   **[P1] 구글 소셜 로그인 기반의 폐쇄형 소통 공간 (방명록/댓글)**
    *   로그인한 사용자만 글을 쓸 수 있음.
    *   작성자 표기 방식: 로그인한 사용자의 `아이디 (실명)` 형태로 강제 표기하여 신원 확인.
*   **[P2] Google API 중심의 확장 기능**
    *   **Google Drive API 연동:** 로컬에 파일을 올리지 않고 구글 드라이브에 올린 나의 PDF 이력서/포트폴리오 파일을 사이트로 곧바로 끌고 와서 노출 및 다운로드 제공.
    *   **Google Analytics 연동:** 포트폴리오에 어떤 교수님이나 채용 담당자가 언제 들어왔는지 나 혼자 볼 수 있는 방문 통계 수집.

### 2.2 선택 및 심화 기능 (Optional)
*   **[P4] 소소한 마이크로 인터랙션** (버튼 호버 시 위로 튀어 오르는 듯한 등 물리적 탄성 애니메이션)

## 3. 기능 상세 설명 및 UI/UX 포커스

### 3.1 인터랙티브 네오 브루탈리즘
*   눈에 띄는 노란색, 시안색, 분홍색 등을 활용하고 요소끼리의 영역 구분을 명확한 검정 테두리로 지침으로써 '투박하지만 세련된' 개발자 포트폴리오만의 감성을 구축합니다.

### 3.2 '신원 확인' 기반의 안전한 소통 (방명록)
*   "누가 글을 쓰거나 사용하는지 알고 싶다"는 요구에 맞추어, 글 작성 시 반드시 구글 로그인을 거치게 합니다. 화면에 노출될 때는 `dev_hong (홍길동)`과 같이 표기되도록 데이터를 가공해 익명성 오남용을 막습니다.

### 3.3 스마트한 파일 및 트래픽 관리 (구글 생태계)
*   이력서가 업데이트 될 때마다 코드를 수정할 필요 없이, 구글 드라이브의 PDF 파일만 갈아끼우면 내 사이트에서도 갱신된 파일이 다운로드 되게끔 똑똑한 구조를 설계합니다.
