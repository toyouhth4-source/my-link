# "마이링크" 사용자 시나리오 (User Scenario)

## 📌 작성 형식 기준
요구사항에 따라 모든 행동 정의는 **"사용자는 [목적]하기 위해 [행동]한다"** 의 형식을 엄격하게 준수하여 작성되었습니다. (v1.3 기획안 기준)

## 시나리오 1. 방문자 (교수님, 학우, 채용 담당자)
**목적:** 하태현 학생의 포트폴리오를 평가하거나 학업 성취도를 파악하고 소통함.

*   사용자는 **하태현의 신원을 3초 만에 파악하기 위해** 전달받은 링크표에 접속하여 프로필 사진과 핵심 텍스트를 확인한다.
*   사용자는 **마우스를 이리저리 움직일 필요 없이 원하는 프로젝트나 이력서를 최단 시간에 찾기 위해** 키보드 단축키 `Ctrl + K` (또는 `Cmd + K`)를 눌러 화면 한가운데에 커맨드 팔레트 검색창을 띄운다.
*   사용자는 **이 학생이 평소에도 코딩을 착실히 하고 있는지 한눈에 평가하기 위해** 프로필 컴포넌트 근처에 배치된 Github 잔디(Contribution) 그래프를 본다.
*   사용자는 **현재 이 학생이 다룰 수 있는 기술 목록을 흐름으로 파악하기 위해** 전광판처럼 끝없이 흘러가는 무빙 텍스트(Marquee) 띠를 주시한다.
*   사용자는 **하태현이 진행했던 과제 결과물을 상세히 살펴보기 위해** 아카이브 보드에 나열된 프로젝트 카드들을 클릭하여 외부 링크로 이동한다.
*   사용자는 **오프라인에서도 포트폴리오를 편안하게 검토하기 위해** 화면 중앙에 배치된 '이력서 다운로드' 버튼을 클릭한다.
*   사용자는 **포트폴리오에 대한 응원이나 평가 피드백을 남기기 위해** 스크롤 맨 하단의 방명록 영역에서 구글 로그인을 거친 후 본인의 실명으로 댓글을 등록한다.
*   사용자는 **어쩌다 실수로 잘못된 하위 주소(예: mylink.com/error)로 접속했을 때 흥미를 잃고 끄는 대신 재미를 느끼기 위해** 해커 느낌의 위트 있는 일러스트가 그려진 404 에러 안내 페이지를 마주한다.

## 시나리오 2. 소유자 (하태현)
**목적:** 주기적으로 자신의 이력과 새로운 프로젝트를 손쉽게 업데이트하고 관리함.

*   소유자는 **포트폴리오 사이트의 관리 권한을 얻기 위해** 본인의 소유자 구글 계정으로 로그인하여 화면 우측 상단에 활성화된 '편집 모드'를 클릭한다.
*   소유자는 **이번 학기에 새로 끝마친 팀 프로젝트를 갤러리에 추가하기 위해** '새로운 링크 추가하기'를 누르고 Github URL과 썸네일 이미지를 등록한다.
*   소유자는 **가장 자신 있는 주요 프로젝트가 교수님 눈에 먼저 띄게 만들기 위해** 프로젝트 카드를 마우스로 꾹 누른 채 드래그 앤 드롭하여 가장 위쪽으로 순서를 끌어올린다.
*   소유자는 **나의 포트폴리오를 오늘 몇 명의 학우가 열람했는지 파악하기 위해** 관리자 메뉴에서 연동된 '방문자 통계(Google Analytics)' 대시보드 버튼을 클릭한다.
*   소유자는 **다음 학기가 되어 변경된 소속을 반영하기 위해** 최상단 프로필 편집 버튼을 누르고 한 줄 소개 텍스트를 수정하여 저장한다.
